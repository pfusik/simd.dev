simd-tooltips demo

Just hover anywhere — when the cursor lands on a SIMD intrinsic name or a SIMD type (like __m256i, int8x16_t, svfloat32_t), a tooltip pops up with its signature and a short description. No DOM walk, no markup mutation: the library detects words under the cursor on the fly using caretPositionFromPoint.

NEON (aarch32 + aarch64)

Loading 8 unsigned 16-bit elements with vld1q_u16, summing two vectors with vaddq_u16, and finding the per-lane maximum with vmaxq_u16:

uint16x8_t add_clamp(const uint16_t *a, const uint16_t *b, uint16_t cap) {
    uint16x8_t va = vld1q_u16(a);
    uint16x8_t vb = vld1q_u16(b);
    uint16x8_t vc = vaddq_u16(va, vb);
    uint16x8_t lim = vdupq_n_u16(cap);
    return vminq_u16(vc, lim);
}

FMA: vfmaq_f32(a, b, c) computes a + b * c per lane on a 4-wide float vector.

SVE / SVE2 (aarch64)

svint32_t saxpy_sve(svbool_t pg, svint32_t x, svint32_t y, int32_t a) {
    return svmla_n_s32_z(pg, y, x, a);
}

svuint8_t lookup(svuint8_t tbl, svuint8_t idx) {
    return svtbl_u8(tbl, idx);
}

The all-lane reduction svaddv_s32 returns a scalar. The predicated load svld1_u8 ignores inactive lanes.

SSE / SSE2 / AVX2 (x86_64)

__m128i sum128(const int32_t *p) {
    __m128i a = _mm_loadu_si128((const __m128i *)p);
    __m128i b = _mm_loadu_si128((const __m128i *)(p + 4));
    return _mm_add_epi32(a, b);
}

__m256i shuffle256(__m256i src, __m256i idx) {
    return _mm256_permutevar8x32_epi32(src, idx);
}

AVX-512 (x86_64)

__m512d add_masked(__m512d src, __mmask8 k, __m512d a, __m512d b) {
    return _mm512_mask_add_pd(src, k, a, b);
}

__m512 fmadd512(__m512 a, __m512 b, __m512 c) {
    return _mm512_fmadd_ps(a, b, c);
}

__mmask16 cmpeq(__m512i a, __m512i b) {
    return _mm512_cmpeq_epi32_mask(a, b);
}

Cross-ISA

The same algorithm in two flavors — note both vfmaq_f32 (NEON) and _mm256_fmadd_ps (Intel FMA) appear inline.

Inline mention

You can also mention vaddq_s8 in prose, or _mm_aesenc_si128, or even an overloaded SVE form like svadd_z (which resolves to many typed variants).
